// 04. Featured Research

Evaluating LLMs in Detecting Spear Phishing Cyberthreats

CYBR 560 Capstone · Hood College · Spring 2026

Python DeepSeek-R1 Llama 4 Scout Qwen 2.5-72B OpenRouter API Groq API Zero-Shot Prompting Excel Analytics
⬡ GitHub Repo

The problem: AI-generated spear phishing campaigns surpassed human red-team attacks in click-through effectiveness for the first time in March 2025 — outperforming expert attackers by 24%. Traditional rule-based email filters were not built for this environment.

The research: A controlled experiment evaluating three freely available open-weight LLMs — DeepSeek-R1, Llama 4 Scout, and Qwen 2.5-72B — across a 205-email dataset including legitimate workplace emails, human-crafted phishing, and AI-generated phishing. Each model used a standardized zero-shot prompt; temperature was set to zero for reproducibility.

The finding: All three models achieved near-perfect phishing detection (≥98.5% recall) but critically high false positive rates on legitimate emails — ranging from 42.7% to 76%. This coexistence prevents standalone deployment and points toward ensemble architecture as the necessary next step.

Key Findings
✓
Near-Perfect Phishing Detection
All three models detected both human-crafted and AI-generated phishing at ≥98.5% accuracy. AI-generated phishing posed no additional detection challenge — a significant finding given it now accounts for ~51% of malicious email volume.
⚠
Critically High False Positives
False positive rates ranged from 42.7% (Qwen) to 76% (Llama). Models systematically misread normal professional email features — urgency language, action requests, links — as threat indicators. Standalone deployment is not operationally viable.
→
Ensemble Architecture Required
High recall + high FPR points to a clear role: these models are strong phishing detection layers within multi-layered ensemble systems, not autonomous classifiers. Hybrid architecture pairing LLM detection with a legitimacy classifier is the recommended path.
